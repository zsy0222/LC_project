# LC_project 极简动画设计方案 v2

> 核心理念：**少即是多**。动画服务于感知而非装饰。每一个动效都回答"发生了什么"。
> 色调体系：**大地色系**——泥土、苔藓、树皮、晨雾。拒绝荧光色。

---

## 〇、全局面板规则

### 色板

```
土壤棕  #5D4037   标题/强调文字
苔藓绿  #2E7D32   主品牌色/按钮/选中态
嫩芽绿  #66BB6A   悬停/高亮/动画
晨雾灰  #F5F5F5   页面底色
云白    #FFFFFF   卡片底色
树皮灰  #9E9E9E   次级文字
暖阳金  #FFB300   奖励/打卡特殊标记
```

### 动画铁律

- 所有过渡 `ease-out`，不弹跳不震荡
- 时长不超过 300ms（除落叶等氛围动画）
- 同一时刻屏幕上最多一个元素在动
- 页面切换用**淡入**（fade），不用位移
- 数字变化用**静默替换**（直接变成新值），不滚动不翻转

### 空间规则

- 卡片间距 10px
- 页面内边距 14px
- 元素圆角 12px（卡片）、10px（按钮）、8px（标签）
- 最大宽度 600px，居中

## 一、登录页 → 选身份

### 1.1 页面加载

```
Timeline:
0ms     背景色 F5F5F5 铺满屏幕
100ms   标题区从 opacity:0 → 1，时长 400ms ease-out
        （只有标题动，图标不动——它一直显示）
300ms   描述文字从 opacity:0 → 1，时长 300ms
500ms   身份卡片从 opacity:0 → 1，时长 300ms
        （卡片不动位置，只变透明度）
800ms   公开排行榜从 opacity:0 → 1，时长 300ms
```

**设计理由**：自上而下逐层出现，模拟"翻看书页"的自然阅读顺序。

### 1.2 选择身份

```
点击下拉框：
  - 下拉框边框颜色过渡：#ddd → #2E7D32，200ms
  - 选项列表从 opacity:0 → 1 + transform: translateY(-4px) → 0
  - 时长 150ms

选中某一项：
  - 下拉框收起的瞬间
  - "进入系统"按钮从 disable 灰色 → 苔藓绿
  - 过渡时长 200ms，颜色渐变（不是瞬切）
  - 按钮可用态增加极微弱的 box-shadow（表示"可以点了"）
```

### 1.3 点击"进入系统"

```
点击瞬间：
  按钮背景加深一瞬（#2E7D32 → #1B5E20），100ms 后恢复
  整个登录页 opacity:1 → 0，时长 250ms ease-out
  
登录页消失后：
  主界面 opacity:0 → 1，时长 250ms ease-out
  整个过程无位移，只有淡入淡出
```

**设计理由**：切换身份是"出发"的动作。按钮变深=按下的触感。页面淡出淡入=场景切换。

## 二、品类选择 → 首次交互

### 2.1 品类卡片初始态

四个品类卡片以 2×2 网格排列，初始状态：

```
卡片默认态：
  - 边框 1px solid #E0E0E0
  - 背景 #FAFAFA（极浅灰，非纯白）
  - 圆角 12px
  - 内边距 12px
  - emoji 32px + 品类名 14px 粗体 + 副标题 11px 灰色
```

### 2.2 品类卡片 — 手指触碰（Hover/Touch）

```
触碰瞬间 (150ms ease-out)：
  - 边框颜色 #E0E0E0 → 对应品类色（非绿色，见下方品类色映射）
  - 卡片背景 #FAFAFA → #FFF（变白=点亮感）
  - 无缩放、无阴影变化（极简原则：只是"亮起来"）

品类色映射：
  外卖厨余 → 暖橙 #FF8F00（食物=暖）
  快递纸箱 → 棕色 #8D6E63（纸箱=棕）
  塑料     → 蓝色 #42A5F5（塑料瓶=蓝）
  有害     → 暗红 #C62828（警示=红）
```

### 2.3 品类卡片 — 点击选中

```
点击瞬间 (200ms ease-out)：
  Step 1: 边框变成品类色，2px 实线（从 1px→2px = 确认感）
  Step 2: 同品类色背景极浅填充（如外卖厨余=#FFF8E1，塑料=#E3F2FD）
  Step 3: 其他三个卡片同时变暗——
          opacity 1 → 0.4，时长 200ms
          （未选中的"退后"，选中的"站出来"）
  Step 4: 选中卡片下方出现品类色的短横线（2px × 24px），
          从 width:0 → 24px，150ms
          （像是下划线标注）
```

**设计理由**：不是"放大选中"，而是"其他退缩"——更安静、更高级。

### 2.4 品类卡片 — 取消选中

```
重新点击已选中的卡片 → 回到默认态，200ms
（暂不做取消功能，但预留逻辑）
```

### 2.5 选择后显示：点位选择区

```
品类选中后 (200ms 延迟)：
  点位选择 section 从 opacity:0 + max-height:0 → opacity:1 + max-height:300px
  时长 300ms ease-out
  （像是展开的折页，从无到有）

  点位卡片从左到右依次出现：
  第1张 延迟 0ms
  第2张 延迟 60ms
  第3张 延迟 120ms
  第4张 延迟 180ms
  每张 opacity:0 → 1，200ms
  （像是依次翻开的手牌）
```

## 三、选点 → 拍照

### 3.1 地点卡片选中

```
逻辑同品类卡片：
  未选中的 → opacity 0.4
  选中的 → 品类色边框 + 品类色浅底
  补充：下方显示 GPS 距离（如果有定位），
        数据出现时带 opacity 过渡 200ms
```

### 3.2 GPS 定位中

```
定位图标：圆环脉冲（纯 CSS，已在用，保持）
文字 "📍 定位中..." 三个点依次出现再消失：
  点1: 0ms → 200ms opacity 1 → 400ms opacity 0
  点2: 200ms → 400ms opacity 1 → 600ms opacity 0
  点3: 400ms → 600ms opacity 1 → 800ms opacity 0
  循环 1 次后停止
```

### 3.3 GPS 定位成功

```
定位成功后：
  - "定位中..." 立即消失
  - "✅ 位置验证通过" Toast 出现（见通用 Toast）
  - 拍照区从 opacity:0 → 1，200ms
  - 拍照区虚线边框做一次极短的呼吸：
    box-shadow: 0 0 0 0 rgba(46,125,50,0) → 0 0 0 4px rgba(46,125,50,0.1) → 0 0 0 0
    时长 600ms，只播一次（引导注意但不反复闪烁）
```

### 3.4 GPS 定位失败

```
失败后：
  - "⚠ 位置偏差约 X 米" Toast 出现（红色文字）
  - 拍照区保持隐藏
  - 提示文字轻微变红 1s 后恢复灰色
```

## 四、拍照 → AI 识别

### 4.1 拍照区（未拍照态）

```
虚线边框 + 相机图标 + "点击上传" 文字
图标做极缓慢的上下浮动：translateY 0 → -4px → 0
周期 3s，ease-in-out 无限循环
（安静的呼吸感，暗示可操作但不催促）
```

### 4.2 点击拍照区

```
点击瞬间 (100ms)：
  - 整个区域 scale(1) → scale(0.98) → scale(1)
  - 像是被轻轻按了一下
```

### 4.3 照片已选择（预览显示）

```
照片从 opacity:0 + filter:blur(4px) → opacity:1 + filter:blur(0)
时长 400ms ease-out
（像是胶片显影——从模糊中浮现）

虚线边框变为实线
上传区背景变为 #FAFAFA
"点击上传"文字消失
```

### 4.4 AI 识别中

```
骨架屏 shimmer：
  识别结果区域出现 3 个灰色矩形条（模拟标签样式的占位）
  每个矩形上叠加一条半透明白色条纹
  条纹从左到右滑动，周期 1.5s，无限循环
  3 个条纹错开 0.3s 相位

同时：拍照区照片保持显示，不加任何动画（静止=在思考）
```

### 4.5 AI 识别完成

```
Shimmer 消失 (100ms)
识别结果分两步出现：
  Step 1 (0ms):    品类标签 + 完整度标签 + 路径标签
                    每个标签从 opacity:0 → 1，延迟 0ms/80ms/160ms
                    （依次弹出，不快不慢）
  Step 2 (200ms):  置信度数字 + 减碳预估数字
                    直接显示，无动画

如果置信度低 (need_recheck)：
  警告条从 opacity:0 → 1，300ms
  警告条做一次极微弱的水平抖动：
  translateX(0 → -2px → 2px → 0)，300ms（"摇头"表示不确定）
```

### 4.6 外卖厨余：拍照后才出现选点区

```
选点区在 AI 结果下方从 opacity:0 → 1，250ms
（和数据标签一样的方式，只是延时到识别完成后触发）
```

