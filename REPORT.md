# 校园低碳回收全路径追踪系统 · 项目报告

## 一、项目概况

**项目名称**：校园低碳回收全路径追踪系统（LC_project）

**项目定位**：面向校园场景的低碳回收闭环 Demo，以 AI 视觉识别为入口、批次化管理为骨架，实现"投递→追踪→去向→成品→反馈"的完整链路。

**技术指标**：Python FastAPI 后端 + 原生 HTML/CSS/JS 单页前端 + SQLite，单文件 SPA，无框架依赖，总代码量约 4800 行。

## 二、核心闭环

```
学生扫码 → 拍照投递 → AI 识别品类/完整度 → 归入批次
    ↓
去向端认领批次 → 上传成品照片
    ↓
投递用户收到反馈（批次故事时间轴 + 成品橱窗）→ 获得碳积分
```

## 四、数据库设计（10 张表）

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `users` | 用户（学生/去向端/管理员） | role, carbon_score, streak |
| `points` | 回收点位 | qr_code, gps_lat, gps_lng |
| `submissions` | 投递记录 | photo, waste_type, batch_id, co2_saved |
| `batches` | 回收批次 | category, status, claimed_by, destination |
| `reuse_items` | 去向端成品 | product_photo, product_desc |
| `activities` | 社团活动 | title, carbon_reward, max_participants |
| `activity_joins` | 活动报名 | uploaded (作品是否已上传) |
| `shop_orders` | 碳积分商城订单 | item_id, address |
| `notifications` | 系统通知 | — |
| `carbon_factors` | 碳排放因子 | co2_per_kg |

核心设计理念：**点位 + 日期 + 品类 = 批次**，同一批次内的投递共享去向端和成品。

## 五、功能模块

### 5.1 投递端（学生视图）

| 功能 | 说明 |
|------|------|
| 扫码签到 | 选择回收点位 → GPS 校验（Haversine 距离）→ 30s 冷却 |
| AI 识别 | MobileNetV3 Small，4 大品类（厨余/纸箱/塑料/有害），支持 mock 降级 |
| 图片去重 | 感知哈希（aHash），跨用户/同点位/同用户三维度去重 |
| 碳积分 | 单次投递减碳 = 品类因子 × 重量估算，20 阶段树木生长 |

### 5.2 去向端（回收者视图）

| 功能 | 说明 |
|------|------|
| 批次抢单 | 按路径分权（如"有机再生"方可见厨余批次） |
| 成品反馈 | 上传改造后成品照片 + 描述 |
| 工艺知识 | 21 种处理工艺知识库（碳减排量 + 原理 + 技术优势 + 数据来源） |

### 5.3 活动广场

社团发布回收主题活动，学生报名参与 → 上传作品 → 获得碳积分（作品上传后才发放）。

### 5.4 碳积分商城

12 件环保商品，碳积分兑换，收货地址管理，已收到确认。

### 5.5 碳积分奖励

- **20 阶段树木生长**：从橡果到参天古树，可视化进度条
- **连续打卡**：7 天×checkin 主题图，成就徽章系统（5 枚）
- **排行榜**：投递次数 × 减碳量排名

### 5.6 批次故事 × 工艺知识（叠卡弹窗）

双卡切换交互设计：前卡展示批次时间线，后卡展示去向工艺知识，28% 露出，点击切换。

## 六、AI 模块

- **模型**：MobileNetV3 Small（ImageNet 预训练权重）
- **输入**：224×224 RGB
- **输出**：Top-5 预测 + 置信度
- **品类映射**：硬编码索引 → 4 大品类
- **完整度评估**：图像统计启发式（HSV 色彩分布 + 边缘密度）
- **降级策略**：`AI_MOCK_MODE=True` 或 torch 导入失败 → 随机结果

## 七、反作弊系统（7 层）

| 层级 | 机制 | 参数 |
|------|------|------|
| 1. GPS 校验 | Haversine 距离 ≤ 100m | `LOCATION_MAX_DISTANCE_M=100` |
| 2. 冷却时间 | 同点位 30s | `COOLDOWN_SECONDS=30` |
| 3. 图片去重-同用户 | 同用户 24h aHash 比对 | `PHOTO_SIMILARITY_THRESHOLD=0.15` |
| 4. 图片去重-同点位 | 同用户同点 7d | `SAME_POINT_DAYS=7` |
| 5. 跨用户去重 | 同点位不同用户 3min | `CROSS_USER_MINUTES=3` |
| 6. 管理员豁免 | `_is_admin()` 跳过全部校验 | — |
| 7. Demo 模式 | `DEMO_MODE=True` 跳过 GPS | 答辩演示用 |

## 八、UI/UX 设计

- **设计系统**：Botanical Journal（纸张/苔藓/墨/树皮/石头/暖色调）
- **字体**：Noto Serif SC（正文 18px）+ Playfair Display（登录标题）
- **登录封面**：极简风，图像居中 + 英文花体 + 毛玻璃卡片点击浮现
- **叠卡弹窗**：非对称边框（右侧 6px bark 粗线 + 左侧 2px 细线），两张卡片大小一致，前后切换
- **响应式**：max-width 600px 移动优先，底栏胶囊式导航

## 九、技术亮点

1. **单文件前端** — 一个 HTML 文件承载全部界面（6 个 Tab + 弹窗系统），零构建工具
2. **轻量 ORM** — SQLAlchemy + 纯 SQLite，零运维，clone 即跑
3. **知识库** — 21 种处理工艺，每条含减排原理、技术优势、数据来源（文献引用）
4. **叠卡交互** — 批次故事与工艺知识双卡设计，80%重叠+28%露出，点击切换
5. **反作弊完备性** — GPS + 感知哈希 + 时间窗口 + 跨用户，多层防御
